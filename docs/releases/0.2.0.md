# FerrisSearch 0.2.0

Release branch: `release/0.2.0`

Comparison base: `v0.0.1..ad33b68`

This release covers 81 commits across 95 files, with 57,435 insertions and 3,275 deletions compared to `v0.0.1`.

## Highlights

- FerrisSearch moved well beyond basic search into a distributed search-plus-analytics engine. The span from `v0.0.1` to `HEAD` adds typed mappings, richer query support, vector search, distributed SQL, performance-focused fast-field execution, operational hardening, and a much stronger test and tooling story.

## Major Changes

- Search and indexing capabilities expanded substantially. The engine gained vector search through the `CompositeEngine`, wildcard/prefix/fuzzy queries, typed field mappings on index creation, field-based sorting with stable tie-breaking, richer aggregations, dynamic settings, and stricter Tantivy type-safety fixes to prevent silent mismatches.
- Replication and cluster correctness improved materially. The release range adds sequence-number ownership in the WAL, ISR checkpoint tracking, replica recovery from the translog, replica promotion, index UUID isolation for on-disk shards, `_count`, better refresh/flush fan-out, `INITIALIZING` shard visibility, and authoritative join snapshots plus hardened node lifecycle behavior around cluster membership.
- SQL became a first-class execution path. This range adds the global SQL endpoint, interactive `ferris-cli`, EXPLAIN and EXPLAIN ANALYZE, distributed grouped partial aggregation, distributed fast-field SQL over Arrow IPC, multi-column GROUP BY, SQL correctness coverage with sqllogictest, planner binder/query-shape validation, same-index semijoins, quoted hyphenated index handling, case-insensitive source-column canonicalization, and end-to-end streamed SQL execution.
- Performance work shifted the system toward search-aware analytics instead of compatibility fallback. Key pieces include FAST-field enablement, LIMIT pushdown, count-star fast paths, skipping `_id` and `_score` reads when not needed, column cache introduction, batch numeric aggregation, direct scan and flat-array grouped aggregation, parallel segment scans, StringArena-based string resolution, deferred JSON materialization, pre-sized coordinator maps, and lazy SQL batch streaming across the coordinator and transport.
- Operability and developer workflow improved across the repo. The diff adds Prometheus metrics, optional feature-gated gRPC TLS, CI automation, large benchmark and ingestion scripts, broader REST and replication coverage, and a substantial README/docs expansion around architecture, benchmarks, dynamic settings, and vector search.

## Notable User-Facing Outcomes

- SQL queries can now stay on fast, search-aware execution paths across multiple nodes instead of falling back to row materialization for many common shapes.
- The CLI is no longer just a thin wrapper around REST calls; it supports interactive SQL workflows, EXPLAIN ANALYZE output, and streamed SQL result consumption.
- Cluster behavior is safer under restart, recovery, and join scenarios because shard data, WAL sequence ownership, and membership transitions are now treated much more explicitly.
